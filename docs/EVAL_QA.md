## 用大白话先讲一遍：什么是"评测"，为什么非做不可

我们做的事是"自动判断合同条款违不违法"。**怎么知道它判得准？** 办法是：拿一批 **标准答案已知** 的题给它做，对一对它错了多少。这就是"评测"。

没有评测，你改完代码根本不知道是变好了还是变坏了——全凭感觉，等于盲开车。

评测要讲清三件事：

1. **标注规范**——怎么出题、怎么定标准答案。
2. **评测集**——题库本身（多少题、什么样）。
3. **评测方法论**——怎么打分、多少分算及格。

下面一件件说。

---

## 1. 标注规范（怎么定标准答案）

每一道题 = 一条合同条款 + 它的标准答案。一道题记这些信息：

| 字段 | 是什么 | 例子 |
|------|--------|------|
| `clause` | 条款原文 | "试用期为 6 个月" |
| `context` | 上下文（合同期限等） | "本合同期限 1 年" |
| `expected.violation` | 违不违法 | true（违法） |
| `expected.category` | 属于哪一类 | probation_period（试用期） |
| `expected.violated_law` | 违反哪条法律 | 劳动合同法 第十九条 |
| `expected.severity` | 严重程度 | high |
| `expected.suggestion` | 该怎么改 | 改到 2 个月以内 |
| `variation_type` | 题型 | violation_clear（明显违法） |

**题型有四种**：明显违法、隐蔽违法（写得像合法）、模糊（边界）、合规。这样能分别看系统在"难题"上表现如何。

**题从哪来 + 怎么保证答案对**：
- 题由大模型 **合成生成**（避免用真人合同的隐私）。
- 答案要 **交叉核查**：用不同的模型互相挑错。比如曾查出并修正 15 处标注错误——其中一条把"恰好 3 个月试用期"误标成违法，其实按民法典"三个月以上"是含 3 个月的，应判合规。

## 2. 评测集（题库长什么样）

| 题库 | 规模 | 说明 |
|------|------|------|
| **文本题库** | 200 条（10 类 × 各 20 条） | 当前主力 |
| **多模态题库** | 扫描 / 拍照 等形态 | ADR-0008 分三阶段，P3 起 |

文本题库的分布：

| 题型 | 数量 |
|------|------|
| 明显违法 | 88 |
| 合规 | 79 |
| 隐蔽违法 | 18 |
| 模糊 | 15 |

违法题占 **52.5%**（这个比例后面算"基线"会用到）。

**合同来源分层**：Tier A 官方范本 / Tier B 网络模板 / Tier C 脱敏真实合同（Tier C 含隐私，不进公开仓库）。

## 3. 评测方法论（怎么打分，多少分及格）

用"**查违法条款**"打个比方：题库 200 条，其中约 104 条真违法。系统跑完，我们看四个分：

| 分数 | 大白话 | 怎么算（例） |
|------|--------|-------------|
| **精确率 Precision（抓得准）** | 你说违法的里面，有多少真违法 | 说了 100 条违法、80 条真违法 → 80% |
| **召回率 Recall（抓得全）** | 真违法的里面，你抓到了多少 | 104 条真违法、抓到 90 条 → 87% |
| **F1（综合分）** | 准和全揉一起的总分 | 越接近 1 越好 |
| **准确率 Accuracy** | 所有题判对的比例 | 200 条对 170 条 → 85% |
| **严重度匹配** | 判对违法后，严重度（高/中/低）也对的比例 | 次要指标 |

补充口径：
- **micro-avg**：把 10 类合起来算一个总分（而不是各类平均）。
- **按题型分别看**：明显 / 隐蔽 / 模糊各自多少分——隐蔽和模糊本来就难，分低正常。
- **基线 0.689**：假设有个"全部判违法的瞎猜检测器"。因为违法题占 52.5%，它全猜违法也能蒙到 F1 ≈ 0.689。**所以真检测器必须明显超过 0.689，才证明它真的有用，而不是在瞎蒙。**
- **各类目的及格线**：每一类在 [taxonomy.yaml](taxonomy.yaml) 里有自己的 `target_precision` / `target_recall`。

---

## 4. 回归门禁 + 质量保障闭环

**门禁**：每次改完代码都跑评测。**不达基线、或某类目不达标 → 不上线。** 这样保证只会越改越好。

**闭环**（多道防线，不只靠评测）：

| 防线 | 防什么 |
|------|--------|
| 评测门禁 | 整体判错 |
| Verifier 复核 | 编造/张冠李戴法条（见 SYSTEM1 §6） |
| 解析 QC | 抽取出错（字符覆盖率 / 乱码率 / 表格完整性，见 SYSTEM1 §3.4） |
| 人工抽检 | 自动指标看不出的问题 |
| 法条库版本化 | 依据过期法律 |
