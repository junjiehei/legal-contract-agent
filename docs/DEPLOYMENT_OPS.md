## 用大白话先讲一遍

这份讲三件事：

1. 系统能 **装在哪**（我们的云、客户的云、还是客户自己内网）。
2. 对隐私敏感的客户（比如律所），怎么做到 **装在内网、数据一步不出门**。
3. 跑起来之后，怎么 **照看**（监控、日志、成本、备份）。

---

## 1. 三种部署模式

| 模式 | 数据放在哪 | 适合谁 | 说明 |
|------|-----------|--------|------|
| **公有云 SaaS** | 我们的云 | 个人用户 | 最方便，开箱即用；数据经脱敏处理 |
| **VPC 专属** | 客户专属云环境 | 中型企业 | 隔离更强，资源独享 |
| **客户内网 on-prem** | 全在客户内网 | 律所 / 大企业 | **数据一步不出门**（ADR-0007 D5） |

越往下，隐私越强、部署越重。三种共用同一套代码，靠配置切换。

---

## 2. 私有化方案（数据不出门怎么做到）

私有部署的难点是"断了外网还得能跑"。做法：

| 要点 | 怎么做 |
|------|--------|
| **不依赖外部 API** | 看图认字的 VLM 改用本地模型，或干脆只用本地 OCR；大语言模型用 **本地 Ollama 跑 DeepSeek** |
| **鉴权** | API Key（HTTP 接口启用） |
| **前端好装** | FastAPI + HTMX **服务端渲染，没有 npm / 没有打包**——一个 Python + 几个 HTML 就能跑（ADR-0010） |
| **一键部署** | Docker 打包 |
| **资源要求** | **CPU 即可，无需 GPU**；内存要够装本地模型（演示机 3.6G 偏紧，私有部署建议配更高内存） |

一句话：**因为前端没有构建链、模型能换本地、用 Docker 打包，所以可以塞进一个断网的内网里跑。**

---

## 3. 运维（跑起来怎么照看）

| 事项 | 做法 | 时机 |
|------|------|------|
| **可观测** | 记录每次调用的 token 数 / 延迟 / 失败用例（Langfuse 或自建） | P4 |
| **审计日志** | 每次 LLM 调用记 hash / 时间 / 成本 / 是否脱敏 | P2 起 |
| **成本追踪** | 每次审查花了多少模型钱（目标单次 < 0.5 元） | P2 起 |
| **缓存** | 相同请求不重复调 LLM，省钱省时 | P4 |
| **多租户隔离** | 不同客户的数据在向量库里分开（Qdrant per-tenant collection） | P4 |
| **法条库更新** | 法律修订时更新条文 + 标注版本 | 持续 |
| **备份** | 向量索引 + 配置定期备份 | 持续 |

---

## 4. 依赖与可达性

- **国内可达**：DeepSeek（国内 API）、PaddleOCR（本地）、Qwen-VL（阿里云）。从国内服务器都连得上。
- **最小依赖**：Python + HTML，无前端构建链，部署门槛低。
- **当前环境**：腾讯云北京六区 / Ubuntu 22.04 / Docker / Qdrant 1.18（已部署）。
